Create Step2Generation.tsx. This component fires the generation request and displays streaming output.

**On mount:**
1. Build the prompt using assembleMasterPrompt-equivalent in renderer (pass all Step 1 selections to IPC)
   - Call window.api.generation.streamContent({ pillar, theme, mechanic, impulse, contentType, customBackgroundPath })
   - The IPC handler (Plan 04) assembles the actual prompt from settings + args
2. Set up listeners:
   - onToken: accumulate into ref, update display state every 100ms via setInterval (anti-flicker per RESEARCH.md pitfall 2)
   - onComplete: call setGenerationComplete(result) in store, clear interval, set isGenerating=false
   - onError: show error message with "Retry" button
3. Clean up listeners on unmount

**Layout:**
- Full-width card showing "Generating your content..." header with animated ellipsis while isGenerating
- Text area showing accumulated tokens (pre-wrapped, monospace or prose font, auto-scroll to bottom)
- Collapsible "View prompt" section (closed by default): shows full assembled prompt text
  - Use a simple show/hide toggle with a chevron icon (no animation needed)
  - Display prompt in a read-only code block
- When isGenerating=false and no error: show "Content ready!" + "Continue to Edit" button
- "Continue" button calls setStep(3) in store
- Error state: amber alert with error message and "Retry" button (retries generation from scratch)
- "New Draft" button visible after completion: calls store.reset() then re-fires generation
