Two bugs cause the hooks hang:

**Bug 1 - Empty prompt sent to Claude API** (`generation.ts` line 52-71):
The `generate:hooks` handler receives `args.prompt` which is always empty string (sent from frontend). The `streamHooks` function forwards this empty string directly to Claude, which errors out.

Fix in `src/main/ipc/generation.ts`:
1. In the `generate:hooks` handler, BEFORE calling `streamHooks`, assemble the prompt from `args.currentHook` and `args.slideContext`:

```typescript
// Build hooks prompt from context
const hooksPrompt = `You are a social media copywriting expert. Generate exactly 3 alternative hook options for an Instagram carousel slide.

Current hook: "${args.currentHook}"

Slide context: ${args.slideContext}

Requirements:
- Each hook must be a different angle/approach
- Keep hooks concise (1-2 sentences max)
- Make them attention-grabbing and scroll-stopping
- Return ONLY a JSON array of 3 strings, no markdown, no explanation

Example format: ["Hook option 1", "Hook option 2", "Hook option 3"]`

// Start streaming (async - don't await)
streamHooks(win, apiKey, hooksPrompt).catch(err => {
  win.webContents.send('generate:error', { message: err.message })
})
```

2. Also in `streamHooks` function: after parsing, strip markdown code fences the same way `streamContent` does (add the `.replace` chain before `JSON.parse`).

**Bug 2 - No onError listener in frontend** (`Step3EditText.tsx` line 122-145):
The `handleRequestHooks` function registers `onHooksComplete` but never registers `onError`. When the API fails, `isLoadingHooks` stays true forever.

Fix in `src/renderer/src/components/wizard/Step3EditText.tsx`:
In `handleRequestHooks`, after the `onHooksComplete` listener registration, add an `onError` listener:

```typescript
const cleanupError = window.api.generation.onError((error) => {
  console.error('Hooks generation failed:', error.message)
  setIsLoadingHooks(false)
  setShowHooksOverlay(false)
  cleanup()
  cleanupError()
})
```

Also update the cleanup in `onHooksComplete` to remove both listeners:
```typescript
const cleanup = window.api.generation.onHooksComplete((result) => {
  setHookOptions(result.hooks)
  setIsLoadingHooks(false)
  cleanup()
  cleanupError()
})
```
