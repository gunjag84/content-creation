  <behavior>
    Balance matrix DB queries (LEARN-01):
    - updateBalanceMatrix(brandId, variableType, variableValue): upserts with INSERT OR REPLACE, increments usage_count, sets last_used to current unix timestamp
    - getBalanceMatrix(brandId): returns all BalanceEntry rows for brand
    - insertSlide(slideData): inserts into slides table, returns slide id
    - updatePostStatus(postId, status): updates posts.status column
    - getPostWithSlides(postId): returns post + array of slides

    Ad-hoc flag (LEARN-05):
    - insertPost already has ad_hoc field
    - getBalanceMatrix filter: when calculating theme balance warnings, filter WHERE ad_hoc = 0

    Warning thresholds (LEARN-03):
    - Mechanic/theme overuse: usage_count > 3 AND last_used within 14 days
    - Warning message format: "{variable_value} used {n}x in {days} days - rotate?"

    Pillar balance (LEARN-06):
    - calculatePillarBalance(balanceEntries, targetPercentages): BalanceDashboardData
    - actual_pct = (pillar_count / total_count) * 100
    - target_pct comes from settings.content_pillars (passed as parameter, not fetched)

    Post IPC channels:
    - 'posts:create' - insertPost(), returns post id
    - 'posts:save-slides' - insertSlide() for each slide, returns slide ids
    - 'posts:update-status' - updatePostStatus()
    - 'posts:get-recommendation-data' - getBalanceMatrix() + generateWarnings() + calculatePillarBalance(), returns { balanceEntries, warnings, dashboardData }
    - 'posts:update-balance' - updateBalanceMatrix() for all variables of a given post
  </behavior>
