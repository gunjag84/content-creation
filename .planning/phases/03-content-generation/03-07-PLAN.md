  <action>
Create Step4RenderReview.tsx.

**On mount:**
- Show "Render & Preview" button (not auto-triggered per CONTEXT.md - user must click)

**Render flow (when user clicks "Render & Preview"):**
1. Load settings via window.api.loadSettings() to get template and brand guidance
2. Load template via window.api.templates.get(templateId from store) to get zones_config and HTML
3. For each slide:
   a. Build slide HTML by injecting slide text into template HTML zones
   b. Apply POST-17: if slide is last slide (cta type), inject brand guidance CTA text/logo from settings.brand_guidance
   c. Set overlay opacity from slide.overlay_opacity
   d. Call window.api.renderToPNG(html, { width: 1080, height: 1350 })
   e. Update progress state: { current: i+1, total: slides.length }
   f. Store returned base64 data URL
4. After all renders: show PNG grid, hide progress
5. Call store.setRenderedPNGs(pngs)

**Progress indicator:**
- Linear progress bar with "Rendering slide 2 of 5..." text
- Each slide shows a small placeholder box that fills with the PNG thumbnail when ready (individual completion feedback)

**Review display:**
- Grid of rendered PNG thumbnails (2-3 per row)
- Each thumbnail: slide number label + type label below image

**Overlay opacity controls (POST-15):**
- Under each PNG thumbnail: Slider (0-100) showing current overlay_opacity * 100
- On slider change: debounce 300ms then:
  a. Update slide.overlay_opacity in store
  b. Re-render only that slide (call renderToPNG again for that slide)
  c. Replace that thumbnail in store.renderedPNGs

**Export flow:**
1. User clicks "Export Feed Post"
2. Call window.api.export.selectFolder() - if canceled, abort
3. Build file list:
   - Each slide PNG: `{date}_{themeSlug}_slide-0{N}.png`
   - caption.txt: `{date}_{themeSlug}_caption.txt` with caption content
4. Call window.api.export.saveFiles(folderPath, files)
5. On success:
   - Update post status to 'approved' via window.api.posts.updateStatus(postId, 'approved')
   - Update balance matrix via window.api.posts.updateBalance(postId, vars)
   - Show "Exported!" toast
   - Advance to Step 5 (Stories)
6. On error: show error toast with details

**Helper: buildThemeSlug(theme: string): string**
- Lowercase, replace spaces with hyphens, remove non-alphanumeric except hyphens
- Example: "Coaching Transformation" -> "coaching-transformation"

**story-generator.ts (main process service):**
Export `buildStoryPrompt(feedPost: { slides, caption, pillar, theme, mechanic }, settings: Settings): string`

Prompt structure:
- Feed post context: share slide texts + caption
- Story types to generate: teaser, reference, deepening, behind_the_scenes
- Story tools catalog: list active tools from settings.story_tools with engagement_type and mechanic_recommendations
- Instructions: generate 2-4 stories, assign story_type and tool_type per story, set timing (teaser/tool recommendation = before, reference/deepening = after)
- Output format: JSON array of StoryProposal objects
- Include system instructions to return ONLY valid JSON

Register in generation.ts IPC handler: the 'generate:stories' handler calls buildStoryPrompt then streams to Claude.

**Step5Stories.tsx:**

- Trigger story generation automatically (user has already exported feed post - stories are the next action)

**On stories complete:**

**Story card display (one card per proposal):**
- Story type badge: teaser/reference/deepening/behind-the-scenes (different colors)

- Story text: editable textarea (bound to local state, not store)

**Edit mode for a story:**
