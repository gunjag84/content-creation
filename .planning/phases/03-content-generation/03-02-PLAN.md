  <behavior>
    Recommendation engine (POST-01, LEARN-04):
    - Cold start (all avg_performance null): returns entry with lowest usage_count per variable_type
    - Tie-breaking on usage_count: alphabetical by variable_value (deterministic)
    - Warm start (any avg_performance non-null): weighted random selection proportional to avg_performance score
    - Warm start fallback: if a variable_type has no entries with avg_performance, use round-robin for that dimension
    - Always returns BalanceRecommendation with reasoning field set correctly
    - brandId defaults to 1

    Prompt assembler (POST-06):
    - Includes brand voice section always
    - Includes target persona section always
    - Includes content pillar and theme section always (uses passed pillar/theme args)
    - Includes mechanic rules when mechanic found in settings.post_mechanics with active=true
    - Includes content defaults (slide count, caption chars, hashtags)
    - Skips competitor_analysis section when field is empty/whitespace
    - Skips viral_post_expertise section when field is empty/whitespace
    - Appends impulse as "## Additional Guidance" when non-empty
    - Always ends with master_prompt_template
    - Estimates token count (chars/4) and truncates optional sections if total > 8000 tokens
    - assembleStoryPrompt: accepts GenerationResult + StoryProposal context, builds story-specific prompt asking Claude for story text/tool content
  </behavior>
