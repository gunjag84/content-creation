The drag-and-drop reorder snaps back because dnd-kit IDs are index-based (`slide-${index}`). After reorder, the SortableContext items array regenerates identical strings, so dnd-kit sees no change.

**Step A - Add uid to Slide type** in `src/shared/types/generation.ts`:
Add `uid: string` field to the `Slide` interface (after `id?: number`). This is a required field.

**Step B - Assign UIDs at generation time** in `src/renderer/src/components/wizard/Step2Generation.tsx`:
In the manual mode `useEffect` (line 38-44), add `uid: crypto.randomUUID()` to each empty slide object.

In the `setGenerationComplete` result processing, slides come from the Claude API `GenerationResult` which does NOT have `uid`. The `setGenerationComplete` action in the Zustand store maps `GenerationResult` slides to `Slide` objects. Find where this mapping happens in `useCreatePostStore.ts` and add `uid: crypto.randomUUID()` there.

**Step C - Use uid for dnd-kit** in `src/renderer/src/components/wizard/Step3EditText.tsx`:

1. `SortableThumbnail` component (line 39): Change `useSortable({ id: \`slide-${index}\` })` to `useSortable({ id: slide.uid })`.

2. `SortableContext` items (line 237): Change `items={generatedSlides.map((_, idx) => \`slide-${idx}\`)}` to `items={generatedSlides.map(s => s.uid)}`.

3. React keys (line 243): Change `key={idx}` to `key={slide.uid}`.

4. `handleDragEnd` (line 100-116): Replace index parsing with uid-based lookup:
```tsx
const handleDragEnd = (event: DragEndEvent) => {
  const { active, over } = event
  if (over && active.id !== over.id) {
    const oldIndex = generatedSlides.findIndex(s => s.uid === active.id)
    const newIndex = generatedSlides.findIndex(s => s.uid === over.id)
    if (oldIndex !== -1 && newIndex !== -1) {
      reorderSlides(oldIndex, newIndex)
      if (activeSlideIndex === oldIndex) {
        setActiveSlideIndex(newIndex)
      } else if (activeSlideIndex === newIndex) {
        setActiveSlideIndex(oldIndex > newIndex ? activeSlideIndex + 1 : activeSlideIndex - 1)
      }
    }
  }
}
```

5. `SortableThumbnail` props interface: Add `slide` already has `Slide` type which now includes `uid`. No change needed to props.
