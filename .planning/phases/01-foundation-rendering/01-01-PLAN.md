       ```
       npm install better-sqlite3 zod zustand
       ```

<task type="auto">

  <files>

  <action>
    1. Define IPC type contracts in src/preload/types.ts:
       ```typescript
       export interface IElectronAPI {
         // Settings
         loadSettings: () => Promise<Settings>
         saveSettings: (settings: Settings) => Promise<void>
         // Database
         getDbStatus: () => Promise<{ ok: boolean; tables: number }>
         // Rendering
         renderToPNG: (html: string, dimensions: { width: number; height: number }) => Promise<string>
         // Security
         saveAPIKey: (key: string) => Promise<void>
         loadAPIKey: () => Promise<string | null>
         deleteAPIKey: () => Promise<void>
         // App
         getAppInfo: () => Promise<{ version: string; userData: string }>
       }
       declare global {
         interface Window { api: IElectronAPI }
       }
       ```
       Use placeholder types for Settings (will be replaced with Zod-inferred type in Plan 02).

    2. Create preload bridge in src/preload/index.ts:
       - Import contextBridge and ipcRenderer from electron
       - Map each IElectronAPI method to ipcRenderer.invoke with namespaced channels (settings:load, settings:save, db:status, render:to-png, security:save-key, security:load-key, security:delete-key, app:info)
       - Expose via contextBridge.exposeInMainWorld('api', api)

    3. Update src/main/index.ts:
       - Create main BrowserWindow with webPreferences pointing to preload script
       - Set dark background color for window (#0f172a slate-900)
       - Register a stub app:info IPC handler that returns version and userData path
       - Stub all other IPC handlers (settings:*, db:*, render:*, security:*) that return placeholder responses so the app does not crash when renderer calls them - these will be replaced by real implementations in Plans 02 and 03
       - Load renderer index.html

    4. Create Sidebar component (src/renderer/src/components/layout/Sidebar.tsx):
       - Left sidebar with collapsible state (icon-only when collapsed)
       - Navigation items: Dashboard (LayoutDashboard icon), Create Post (PenSquare icon), Settings (Settings icon) - use lucide-react icons
       - Active state highlighting
       - Collapse toggle button at bottom
       - Dark theme styling with slate colors
       - Do NOT use shadcn/ui Sidebar component (overly complex for our needs). Build a simple custom sidebar with Tailwind classes.

    5. Create AppLayout component (src/renderer/src/components/layout/AppLayout.tsx):
       - Flex container: Sidebar on left, main content area on right
       - Main content area with padding and overflow-auto

    6. Create placeholder pages:
       - Dashboard.tsx: Simple welcome page with app name "Content Creation System", brief description, status indicators for DB and settings (will connect to real data in Plan 02)
       - TestRender.tsx: Placeholder with "Rendering Test" heading and description text explaining this will be the rendering test page (actual render button added in Plan 03)

    7. Create App.tsx with simple state-based routing (useState for current page):
       - Wrap in AppLayout with Sidebar
       - Render Dashboard or TestRender based on active nav item
       - Dark mode class on root element (add "dark" class to html in index.html)

    8. Update src/renderer/index.html: Add class="dark" to html element for dark mode default.

    9. Verify app launches with `npx electron-vite dev` (it should show the dark-themed sidebar with navigation).
  </action>
