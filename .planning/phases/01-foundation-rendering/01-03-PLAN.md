    1. Create src/main/services/render-service.ts:
       - class RenderService with private renderWindow: BrowserWindow | null
       - initialize(): Create BrowserWindow with:
         - show: false (CRITICAL - NOT hide(), per research pitfall #2)
         - width: 1080, height: 1920 (max needed dimension - story size)
         - frame: false
         - webPreferences: { offscreen: false, nodeIntegration: false, contextIsolation: true }
         - Load blank HTML page: data:text/html,<html><body></body></html>
         - Wait for did-finish-load event before resolving

       - renderToPNG(html: string, dimensions: { width: number; height: number }): Promise<string>
         - Throw if renderWindow not initialized
         - Set window size to dimensions.width x dimensions.height
         - Load HTML via loadURL with data: URI (encodeURIComponent the HTML)
         - Wait for did-finish-load event
         - Add 150ms delay after load for CSS rendering (per research recommendation, slightly higher than 100ms for safety)
         - Call webContents.capturePage() to get NativeImage
         - Save image.toPNG() to temp directory using app.getPath('temp')
         - Filename format: render_{timestamp}_{width}x{height}.png
         - Return the full file path

       - renderCarousel(slides: string[], dimensions: { width: number; height: number }): Promise<string[]>
         - Convenience method: render each slide sequentially, return array of paths
         - Sequential rendering in same window (per CONTEXT.md decision)

       - cleanup():
         - Close renderWindow if not null, set to null

       - Export registerRenderingIPC(renderService: RenderService)
       - Register ipcMain.handle('render:to-png', async (event, html, dimensions) => renderService.renderToPNG(html, dimensions))
