## Decisions Made

- Parameter-based `buildSlideHTML` instead of closure over component state - makes it testable and callable from any context (Step 3, Step 4, future editor panels)
- Nullish coalescing for override merge (`override.x ?? zone.x`) - correctly handles explicit 0 values that falsy checks would miss
- `buildSlideHTMLForIndex` wrapper in Step4 instead of inline call restructuring - minimal diff, preserves existing call sites
- `canUndo`/`canRedo` as functions using `getState()` - consistent with Zustand pattern, avoids stale closure issues
