<task type="auto">
  <name>Task 1: Fix Switch component styling for Tailwind v4 dark theme</name>
  <files>src/renderer/src/components/ui/switch.tsx, src/renderer/src/styles/globals.css</files>
  <action>
  **Diagnosis:** The project uses Tailwind CSS v4 (`@import "tailwindcss"`) which changed how custom colors work. The shadcn Switch component uses classes like `bg-primary`, `bg-input`, `bg-background` which rely on a theme mapping from CSS variables to Tailwind utilities. In Tailwind v4, these need to be registered with `@theme` directive in the CSS or use explicit `bg-[hsl(var(--primary))]` syntax.

  **Fix approach - two parts:**

  **Part A: Update globals.css to register theme colors with Tailwind v4:**
  Add a `@theme` block that maps CSS variables to Tailwind color utilities:
  ```css
  @theme {
    --color-background: hsl(var(--background));
    --color-foreground: hsl(var(--foreground));
    --color-card: hsl(var(--card));
    --color-card-foreground: hsl(var(--card-foreground));
    --color-popover: hsl(var(--popover));
    --color-popover-foreground: hsl(var(--popover-foreground));
    --color-primary: hsl(var(--primary));
    --color-primary-foreground: hsl(var(--primary-foreground));
    --color-secondary: hsl(var(--secondary));
    --color-secondary-foreground: hsl(var(--secondary-foreground));
    --color-muted: hsl(var(--muted));
    --color-muted-foreground: hsl(var(--muted-foreground));
    --color-accent: hsl(var(--accent));
    --color-accent-foreground: hsl(var(--accent-foreground));
    --color-destructive: hsl(var(--destructive));
    --color-destructive-foreground: hsl(var(--destructive-foreground));
    --color-border: hsl(var(--border));
    --color-input: hsl(var(--input));
    --color-ring: hsl(var(--ring));
    --radius-sm: calc(var(--radius) - 4px);
    --radius-md: calc(var(--radius) - 2px);
    --radius-lg: var(--radius);
  }
  ```

  **Part B: If the @theme approach doesn't resolve the classes** (test by building), fall back to explicit inline color values in the Switch component:
  Replace `bg-primary` with `bg-[hsl(var(--primary))]` and `bg-input` with `bg-[hsl(var(--input))]`:
  ```typescript
  "data-[state=checked]:bg-[hsl(var(--primary))] data-[state=unchecked]:bg-[hsl(var(--input))]"
  ```
  And for the thumb: `bg-[hsl(var(--background))]`

  **Also verify the switch is visually distinct in dark mode:**
  - Checked state: should show blue/primary colored track
  - Unchecked state: should show dark gray/input colored track
  - The thumb (circle) should be clearly visible against both states

  After fixing CSS, verify by checking that MechanicsSection and StoryToolsSection toggles:
  1. Are visually visible (not invisible/transparent)
  2. Show visual state change when clicked (track color changes)
  3. The click event fires (add a console.log temporarily if needed, remove after)
