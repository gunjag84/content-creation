    **Add constant after existing constants (after line 34):**


    **Fix initial containerSize state (line 66):**
    Change:
    ```typescript

    ```
    To:
    ```typescript

    ```

    **Fix scale computation (line 76):**
    The current `scale` is only width-based: `const scale = containerSize.width / CANVAS_WIDTH`

    Change to use the constrained height from state (which is already capped in the ResizeObserver):
    ```typescript
    const scale = containerSize.width / CANVAS_WIDTH
    ```
    This line can stay - the fix is in the ResizeObserver below that correctly computes both width and height to derive scale.

    **Fix ResizeObserver callback (lines 82-88):**
    Change:
    ```typescript
    const resizeObserver = new ResizeObserver((entries) => {
      const entry = entries[0]
      if (entry) {
        const width = entry.contentRect.width
        const height = (width / CANVAS_WIDTH) * canvasHeight
        setContainerSize({ width, height })
      }
    })
    ```
    To:
    ```typescript
    const resizeObserver = new ResizeObserver((entries) => {
      const entry = entries[0]
      if (entry) {
        const width = entry.contentRect.width
        // Compute scale limited by BOTH width and height constraints
        const scaleByWidth = width / CANVAS_WIDTH
        const scaleByHeight = MAX_DISPLAY_HEIGHT / canvasHeight
        const constrainedScale = Math.min(scaleByWidth, scaleByHeight)
        const constrainedWidth = CANVAS_WIDTH * constrainedScale
        const constrainedHeight = canvasHeight * constrainedScale
        setContainerSize({ width: constrainedWidth, height: constrainedHeight })
      }
    })
    ```

    **Also fix the scale variable** - since containerSize.width is now the DISPLAY width (not container width), and scale = containerSize.width / CANVAS_WIDTH already gives the correct scale (because containerSize.width = CANVAS_WIDTH * constrainedScale), this is naturally correct.

    **Fix canvas container div (line 380):**
    The container div uses `max-w-[700px]` but the canvas itself dictates the actual size. Add explicit width/height inline styles to match containerSize so the Stage renders correctly:
    ```tsx
    <div
      ref={containerRef}
      className="border border-slate-700 rounded-lg overflow-hidden bg-slate-900 mx-auto"
      style={{
        width: containerSize.width,
        height: containerSize.height,
        cursor: drawMode ? 'crosshair' : 'default'
      }}
    >
    ```
    Remove the `max-w-[700px]` class since the containerRef-based ResizeObserver now controls sizing.
