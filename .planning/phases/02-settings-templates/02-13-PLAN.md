<task type="auto">
  <name>Task 2: Fix Button outline variant for dark context</name>
  <files>src/renderer/src/components/ui/button.tsx</files>
  <action>
    The current outline variant is:
    ```
    'border border-slate-200 bg-white hover:bg-slate-100 hover:text-slate-900': variant === 'outline',
    ```

    `bg-white` makes the button invisible against dark panels. The text (which defaults to inherited color - in dark context that's slate-100) becomes unreadable against white.

    Change outline variant to be transparent with explicit dark-context text color:
    ```
    'border border-slate-600 bg-transparent text-slate-200 hover:bg-slate-700 hover:text-slate-100': variant === 'outline',
    ```

    This ensures outline buttons work in dark context (the primary use case in this app). The default/ghost variants stay unchanged.

    Note: This change affects ALL outline buttons app-wide. Verify that no light-context outline buttons exist by checking if Button outline is used outside template builder and zone editor. If it's used in TemplateBuilder.tsx wrapping dark context, this is safe. The only potential concern is if outline buttons appear on white/light backgrounds - search for other usages.

    After making the change, also update the base class `ring-offset-white` to `ring-offset-slate-900` since the app is dark-themed throughout. This prevents a white ring-offset flash on focus.
  </action>
  <verify>npm run typecheck 2>&1 | tail -5</verify>
