<task type="auto">
  <name>Task 1: Fix render-service.ts to use temp file instead of data URI</name>
  <files>src/main/services/render-service.ts</files>
  <action>
    The current implementation encodes HTML as a data: URI for loadURL. Chromium has a ~2MB limit on data: URI length. Brand preview HTML with embedded font-face declarations easily exceeds this. Fix: write HTML to a temp file and load via file:// protocol.

    Replace the renderToPNG method's HTML loading section. Change from:
    ```
    const encodedHTML = encodeURIComponent(html)
    const dataURI = `data:text/html,${encodedHTML}`
    await new Promise<void>((resolve, reject) => {
      this.renderWindow!.webContents.once('did-finish-load', () => resolve())
      this.renderWindow!.loadURL(dataURI).catch(reject)
    })
    ```

    To:
    ```
    // Write HTML to temp file to avoid data: URI length limits (~2MB cap in Chromium)
    const timestamp = Date.now()
    const htmlFilename = `render_${timestamp}.html`
    const htmlFilePath = path.join(tempDir, htmlFilename)
    fs.writeFileSync(htmlFilePath, html, 'utf-8')

    await new Promise<void>((resolve, reject) => {
      this.renderWindow!.webContents.once('did-finish-load', () => resolve())
      this.renderWindow!.loadFile(htmlFilePath).catch(reject)
    })
    ```

    Note: `loadFile` is the correct Electron API for loading local files (vs `loadURL` with a file:// string). It handles path encoding correctly cross-platform.

    Also: add cleanup of the temp HTML file after capture to avoid accumulation:
    ```
    // Capture the page as PNG
    const image = await this.renderWindow.webContents.capturePage()

    // Cleanup temp HTML file
    try { fs.unlinkSync(htmlFilePath) } catch (_) { /* ignore */ }
    ```

    Move `const tempDir = app.getPath('temp')` before the HTML file write (it's currently declared after the loadURL block - it needs to come earlier now). Ensure the final PNG filename still uses timestamp and dimensions. The return value JSON format stays the same.

    Also increase the CSS rendering delay from 150ms to 300ms to ensure fonts load from file:// references before capture.
  </action>

  <done>TypeScript compiles. render-service.ts uses fs.writeFileSync + loadFile for HTML loading. Temp HTML file is cleaned up after capture.</done>
