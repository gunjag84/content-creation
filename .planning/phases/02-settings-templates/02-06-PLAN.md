**1. CarouselVariantEditor.tsx (TPL-09):**

Per requirement: "Templates support carousel variants (separate cover slide, content slide, CTA slide HTML with shared config)."

Carousel templates need different zone layouts for different slide types:
- **Cover slide**: Usually has a hook zone prominently, minimal body text
- **Content slides**: Typically have body zones for the main content
- **CTA slide**: Has CTA zone, logo, handle - follows lastSlideRules from brand guidance

```typescript
interface CarouselVariantEditorProps {
  variants: CarouselVariants
  onChange: (variants: CarouselVariants) => void
  brandGuidance: VisualGuidance
  backgroundImage: HTMLImageElement | null
}

interface CarouselVariants {
  cover: Zone[]    // zones for the first slide
  content: Zone[]  // zones for middle slides (reused across all content slides)
  cta: Zone[]      // zones for the last slide
}
```

Integration with TemplateBuilder (Plan 05):
- When format is "feed" AND user enables "Carousel Mode" toggle, show the CarouselVariantEditor
- Three tabs within the editor: "Cover Slide", "Content Slides", "CTA Slide"
- Each tab shows the ZoneEditor (from Plan 05) with zones specific to that variant
- Shared settings: background, overlay settings apply to all variants
- zones_config in the database stores the variants as JSON:
  ```json
  { "type": "carousel", "cover": [...], "content": [...], "cta": [...] }
  ```
- Non-carousel templates store zones as a flat array (backward compatible)

If this integration requires modifying TemplateBuilder.tsx (Plan 05), make minimal changes:
- Add a "Carousel Template" toggle
- When enabled, replace single ZoneEditor with CarouselVariantEditor
- Update save logic to store variant-aware zones_config
