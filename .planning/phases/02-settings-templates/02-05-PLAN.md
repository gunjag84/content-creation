**1. ZoneEditor.tsx - Canvas-based zone editor using react-konva:**

Per locked decision: "Click-and-drag freeform rectangle drawing on uploaded image preview. Resize handles shown on selection. Zone types: hook, body, CTA, no-text."

```typescript
interface ZoneEditorProps {
  backgroundImage: HTMLImageElement | null
  backgroundType: 'image' | 'solid_color' | 'gradient'
  backgroundColor?: string
  overlayColor: string
  overlayOpacity: number
  overlayEnabled: boolean
  zones: Zone[]
  onZonesChange: (zones: Zone[]) => void
  brandGuidance: VisualGuidance  // for font sizes and colors
  format: 'feed' | 'story'  // determines canvas aspect ratio
}
```

Canvas setup:
- Full canvas dimensions: 1080x1350 (feed) or 1080x1920 (story)
- Scale canvas to fit container width (~600-700px) using a scale factor
- Store zones in 1080-scale coordinates (not display coordinates) for rendering consistency
- Use ResizeObserver on container to handle window resize (see RESEARCH pitfall 4)

Drawing new zones:
- When user clicks empty canvas area and drags, draw a new Rect
- Track mousedown position, calculate width/height on mousemove, finalize on mouseup
- New zone gets a generated UUID, defaults to type 'body', default font sizes from brandGuidance
- Minimum zone size: 50x50px (at 1080 scale)

Selecting and manipulating zones:
- Click on existing zone to select it (show Transformer with resize handles)
- Click on empty canvas area to deselect
- Selected zone is draggable within canvas bounds
- Transformer enables resize from any edge/corner
- On drag end: update zone x, y in state
- On transform end: read scale factors, apply to width/height, reset scale to 1 (see RESEARCH pattern)
- Bound zone movement to canvas edges (prevent dragging off-screen)

Zone visual styling by type:
- hook: blue overlay (rgba(59, 130, 246, 0.3)), blue border
- body: green overlay (rgba(34, 197, 94, 0.3)), green border
- cta: orange overlay (rgba(249, 115, 22, 0.3)), orange border
- no-text: red overlay (rgba(239, 68, 68, 0.3)), red dashed border

Per locked decision (template live preview): "Sample placeholder text renders inside zones as user draws/resizes them. Uses actual brand fonts and standard font sizes."

Render sample text inside each zone:
- hook zone: "Your hook text here" in headline font
- body zone: "Body text content goes in this area" in body font
- cta zone: "Call to action" in CTA font
- no-text zone: Show "No Text Zone" label only (no content)
- Use Konva Text nodes inside each zone rectangle
- Text auto-sizes to fit zone (use fontSize from zone config, show approximate character count)

Overlay layer (between background and zones):
- When overlayEnabled: render a full-canvas Rect with overlayColor and overlayOpacity
- This shows the readability impact of the overlay on the background

Delete zone:
- "Delete" button in zone popover, or keyboard shortcut (Delete/Backspace when zone selected)

**2. ZonePopover.tsx - Inline configuration popover:**

Per locked decision: "Inline popover appears when clicking a zone. Shows dropdowns for type assignment, font size settings."

```typescript
interface ZonePopoverProps {
  zone: Zone
  position: { x: number; y: number }  // screen position for popover
  onUpdate: (updates: Partial<Zone>) => void
  onDelete: () => void
  brandGuidance: VisualGuidance
}
```

- Position the popover near the selected zone (above or below, whichever has more space)
- Show:
  - Zone type selector (select/dropdown): hook, body, CTA, no-text
  - Font size display (auto-determined from brand guidance based on type: hook -> headlineFontSize, body -> bodyFontSize, cta -> ctaFontSize)
  - Min font size input (fallback for auto-shrink)
  - Approximate character count (calculated from zone dimensions and font size)
  - "Delete Zone" button (destructive, red)
- When zone type changes, auto-update fontSize to match the brand guidance default for that type
- Use `event.stopPropagation()` on popover interactions to prevent closing issues (see RESEARCH pitfall 6)

**Drawing mode UX:**
- Default mode: "Select" (click zones to select/drag/resize)
- "Draw Zone" button activates draw mode (cursor changes to crosshair)
- After drawing a zone, auto-select it and show the popover for type assignment
- Drawing mode deactivates after zone is drawn (returns to Select mode)

**Toolbar above canvas:**
- "Draw Zone" button (toggles draw mode)
- "Delete Selected" button (enabled when zone selected)
- Zone count indicator: "3 zones defined"

**3. TemplateBuilder.tsx - Full template creation flow:**

Orchestrates the complete template creation experience:

```typescript
interface TemplateBuilderProps {
  templateId?: number  // if editing existing template
  onSave: (templateId: number) => void
  onCancel: () => void
}
```

Layout:
```
+--------------------------------------------------+
| Template Builder                                   |
| Name: [___________]  Format: (Feed) (Story)       |
+--------------------------------------------------+
| [Background Selector]                              |
+--------------------------------------------------+
| Toolbar: [Draw Zone] [Delete Selected] | 3 zones  |
| +----------------------------------------------+  |
| |                                              |  |
| |           Zone Editor Canvas                 |  |
| |           (background + overlay + zones)     |  |
| |                                              |  |
| +----------------------------------------------+  |
| [Overlay Controls]                                 |
+--------------------------------------------------+
| [Cancel]                              [Save]       |
+--------------------------------------------------+
```

State management:
- Template name (Input field, required)
- Format (radio: feed 1080x1350 / story 1080x1920)
- Background type + value
- Overlay settings (enabled, color, opacity)
- Zones array

Background image handling:
- When user selects "Image" background and uploads, store the image in `userData/templates/images/` via IPC
- Create an IPC handler or reuse the logo upload pattern for template background images
- Load image as HTMLImageElement for the Konva canvas

**Save flow:**
1. Validate: name is not empty, at least one zone defined (unless user confirms "save template with no zones?")
2. Prepare template data matching TemplateInsert schema
3. zones_config: JSON.stringify(zones)
4. background_value: file path for image, hex color for solid, CSS gradient string for gradient
5. Call `window.api.templates.create(templateData)` or `window.api.templates.update(id, templateData)` if editing
6. Call onSave with returned template ID

**Edit mode:**
- If `templateId` is provided, load existing template via `window.api.templates.get(templateId)`
- Pre-populate all fields from loaded template
- Parse zones_config JSON back into Zone[]
- "Save" calls update instead of create

Format switching:
- Changing format updates canvas dimensions (1080x1350 vs 1080x1920)
- Warn user if zones exist (they may go out of bounds on format change)
