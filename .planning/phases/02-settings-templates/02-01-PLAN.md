**1. Add template query functions to `src/main/db/queries.ts`:**

```typescript
// Template types
export interface TemplateInsert {
  brand_id?: number
  name: string
  background_type: 'image' | 'solid_color' | 'gradient'
  background_value: string
  overlay_color?: string
  overlay_opacity?: number
  overlay_enabled?: boolean
  format: 'feed' | 'story'
  zones_config: string // JSON string of Zone[]
}

export interface Template extends TemplateInsert {
  id: number
  created_at: number
  updated_at: number
}

// Template CRUD
export function insertTemplate(data: TemplateInsert): number
export function getTemplate(id: number): Template | undefined
export function listTemplates(brandId?: number): Template[]
export function updateTemplate(id: number, data: Partial<TemplateInsert>): void
export function deleteTemplate(id: number): void
export function duplicateTemplate(id: number, newName: string): number
```

Implement each using prepared statements matching the existing query patterns in queries.ts. `duplicateTemplate` reads the source template and inserts a copy with the new name.

**2. Add settings version query functions to `src/main/db/queries.ts`:**

The `insertSettingsVersion` function already exists from Phase 1. Add these read queries needed by the Settings History UI (Plan 02-02 Task 3):

```typescript
// Settings version types
export interface SettingsVersion {
  id: number
  brand_id: number
  filename: string
  timestamp: number
}

// List all settings versions for a brand, newest first
export function listSettingsVersions(brandId?: number): SettingsVersion[]

// Get the settings version that was active at a given timestamp
// (the version with the highest timestamp <= the given timestamp)
export function getSettingsVersionAtTime(timestamp: number, brandId?: number): SettingsVersion | undefined

// Get the settings version associated with a post (via settings_version_id FK)
export function getSettingsVersionForPost(postId: number): SettingsVersion | undefined
```

Implement with prepared statements. `listSettingsVersions` orders by `timestamp DESC`. `getSettingsVersionAtTime` uses `WHERE timestamp <= ? ORDER BY timestamp DESC LIMIT 1`. `getSettingsVersionForPost` joins posts.settings_version_id to settings_versions.id.
