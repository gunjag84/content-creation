# Hook System - Design Doc

Generated by /plan-ceo-review on 2026-03-19
Branch: MECE | Mode: SCOPE EXPANSION

## Problem

Hooks are the single highest-leverage variable in Instagram content. The current system optimizes *what* to post (area/method/tonality balance) but not *how to stop the scroll*. No hook patterns are tracked, no performance feedback loop exists for hooks, and collected hooks from other creators have no structured home.

## Vision

### 10x Version
Hooks as the primary creative lever with their own intelligence loop:
1. Hook Pattern taxonomy + Hook Library (baseline)
2. Cross-dimension performance intelligence ("Your H3 + T4 = 3.4x reach")
3. AI generates pattern-specific hook candidates, user picks, auto-saves to library
4. Proactive hook recommendations based on balance + performance data
5. (Future) Competitor hook scraping, hook A/B testing

### Platonic Ideal
User opens CreatePost. Before thinking about dimensions, the system shows 3 hooks that would diversify the feed and match high-performing patterns. Each shows the pattern, estimated reach multiplier, one-click "use this." The hook library feels like a Pinterest board - visual, browsable, filterable by pattern/source/performance. Every post creation auto-populates the library.

## Architecture

### Two-Layer Design
1. **Hook Patterns** (config) - Stored in `settings.json` alongside areas/methods/tonalities. Schema: `{id, name, description, example}`. Managed via DimensionListEditor in BrandConfig. Required dimension on new posts.
2. **Hook Library** (data) - Stored in `hook_library` SQLite table. Schema: `{id, text, patternId (nullable), source, sourceDetail, post_id (nullable), created_at}`. Grows with usage via auto-save during generation + manual additions. patternId is nullable (orphaned entries when patterns are deleted).

## Accepted Scope

### Core (from concept doc)
- Hook patterns as configurable dimension in settings (like areas/methods)
- Hook library in SQLite table
- Hook pattern selector in CreatePost (required)
- Prompt assembler: hook pattern context injection
- Balance matrix: hook_pattern as tracked variable_type
- Post history: hook pattern badge
- BrandConfig: hook patterns section + hook library browser

### Expansions (CEO review)
1. **AI Hook Generation** - AI generates 3 pattern-specific hook candidates during post creation. User picks one. Selected hook auto-saves to library.
2. **Cross-Dimension Analytics** - After 5+ posts per pattern, show performance insights: "H3 Kontrast avg 2.1x reach" and cross-dimension correlations: "H3 + T4 = 3.4x reach."
3. **Auto-Classify Existing Posts** - Backfill hook_pattern for existing posts by analyzing hook_text against pattern descriptions. Batched (10 per AI call). Review step before committing.
4. **Balance Recommendations** - hook_pattern included in learning-service recommendations and rotation warnings.
5. **Blacklist Rules** - hook_pattern as valid dimension in blacklist rules (e.g., H4 Storytelling + M7 Faktencheck = blocked).
6. **Hook Preview** - When selecting a pattern in CreatePost, show 2-3 example hooks from library (or pattern example if library empty).

## Key Decisions

| Decision | Choice | Reasoning |
|----------|--------|-----------|
| hook_pattern required or optional | Required | Data accumulation for feedback loop is the whole point |
| Hook library storage | SQLite table (not settings.json) | Library is transactional data, not config. Supports pagination/filtering. |
| Pattern deletion + library entries | Orphan (set patternId=null) | Hook text has value independent of classification |
| Auto-classify batching | 10 hooks per AI call | 10x faster than one-at-a-time, same token cost |
